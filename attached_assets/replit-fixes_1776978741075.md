## Fix 1 — UI does not update without page refresh

**Problem:** After any action (delete student, add user, update record...) the UI does not reflect the change immediately. The user has to manually refresh the page.

**Fix:** After every successful mutation (create, update, delete) across ALL pages and components, invalidate and refetch the relevant query so the UI updates instantly. Use `refetch()` or React Query's `invalidateQueries` after every `onSuccess` callback throughout the entire codebase. Apply this to students, users, payments, expenses, and all other data lists.

## Fix 2 — Marketing registration requests behavior

**File:** `artifacts/kidspeak/src/pages/students/index.tsx` (and related components)

**Problem:** When a student request from the marketing section is accepted or rejected, nothing changes visually without a page refresh. Also, accepted students should disappear from the marketing list.

**Fix:**
- When a student request is **accepted**: remove them immediately from the "طلبات التسجيل من التسويق" list and add them to the main students list. Refetch both lists after the action.
- When a student request is **rejected**: move them back to the marketing pipeline with a "rejected" status and remove them from the pending requests view.
- All changes must reflect instantly in the UI without any page refresh.

## Fix 3 — Student status system (Active / Stopped / Graduated)

**Problem:** There is no way to distinguish between students who are currently studying, students who have stopped, and students who have completed the full program.

**Fix:**
1. Add a `status` field to the student schema with three values: `active` (يدرس حالياً), `stopped` (توقف), `graduated` (تخرج).
2. Display a colored status badge on each student card:
   - `active` → green badge
   - `stopped` → orange/yellow badge  
   - `graduated` → blue/purple badge
3. Add a filter dropdown on the students page to filter by status.
4. Allow changing a student's status from their profile or from the student card (dropdown or button).
5. Default status for all existing students should be `active`.

## Fix 4 — Employee salary section

**Problem:** There is no dedicated section to manage and record employee salaries.

**Fix:**
1. Create a salary management section for admins where they can add salary payments for each employee (user).
2. Each salary entry should include: employee name, amount, date, and optional note.
3. When a salary is added:
   - It must appear in the employee's own account under a "راتبي" / "My Salary" section visible when they log in.
   - It must automatically be added to the expenses tracker with the category "salary" / "رواتب".
4. Show salary history per employee with date and amount.
5. Admins should be able to view total salary expenses per month.
