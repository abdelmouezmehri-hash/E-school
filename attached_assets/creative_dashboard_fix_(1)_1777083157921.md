# إصلاح وتطوير المحرك الإبداعي (Studio)

## 1 — حذف قسم التلاميذ من واجهة المبدعين

**المشكلة:**
المبدعون (مصمم، مصور، مسوق) يرون قسم "التلاميذ" في القائمة الجانبية رغم أنه لا علاقة لهم به.

**الإصلاح في `artifacts/kidspeak/src/components/layout.tsx`:**

ابحث عن منطق القائمة الجانبية وأضف شرطاً يُخفي قسم التلاميذ للأدوار الإبداعية:
```ts
const creativeRoles = ["designer", "photographer", "marketer"];
const isCreative = creativeRoles.includes(user.role);

// أخفِ هذه الأقسام للمبدعين:
// - التلاميذ
// - الأفواج
// - التقييمات
// - الأداء
// - المتابعة السلوكية
// - المداخيل
// - الرواتب
```

أضف `&& !isCreative` لكل قسم لا علاقة له بالمبدعين.

## 2 — لوحة التحكم الجديدة للمبدعين

**الملف:** `artifacts/kidspeak/src/pages/studio/index.tsx`

### الأقسام المطلوبة:

**أ) بطاقات إحصائية (أعلى الصفحة):**
```
[المهام النشطة]  [بانتظار المراجعة]  [مكتملة هذا الشهر]  [مستحقاتي]
```
- المهام النشطة: عدد المهام بحالة "للتنفيذ" أو "قيد التنفيذ"
- بانتظار المراجعة: عدد المهام بحالة "في المراجعة"
- مكتملة: عدد المهام بحالة "منشور" للشهر الحالي
- مستحقاتي: إجمالي مستحقات المبدع من المهام المكتملة

**ب) قسم المهام مع Tabs:**
```
[للتنفيذ] [قيد التنفيذ] [في المراجعة] [معتمد] [منشور]
```
لكل مهمة: اسم المهمة، نوعها، الموعد النهائي، المبلغ، زر مناسب للحالة.

**أزرار حسب الحالة:**
- للتنفيذ → زر "بدء التنفيذ" (يغير الحالة إلى "قيد التنفيذ")
- قيد التنفيذ → زر "رفع ملف" + زر "إرسال للمراجعة"
- في المراجعة → "بانتظار الموافقة" (قراءة فقط للمبدع)
- معتمد → للمسؤول فقط: زر "نشر"
- منشور → عرض فقط

**ج) قسم الأصول المرفوعة:**
- عرض الملفات المرفوعة للمهام النشطة
- زر رفع ملف جديد
- عرض: اسم الملف، الحجم، تاريخ الرفع

**د) قسم مستحقاتي:**
- قائمة بكل مهمة مكتملة مع المبلغ وحالة الدفع (مدفوع/معلق)

## 3 — تحديث حالة المهمة

### من طرف المبدع:
| من | إلى | الشرط |
|-----|-----|-------|
| للتنفيذ | قيد التنفيذ | المبدع المُعيَّن فقط |
| قيد التنفيذ | في المراجعة | بعد رفع ملف |

### من طرف المسؤول:
| من | إلى | الشرط |
|-----|-----|-------|
| في المراجعة | قيد التنفيذ | رفض وإعادة للتعديل |
| في المراجعة | معتمد | قبول العمل |
| معتمد | منشور | نشر العمل |

**الإصلاح في `artifacts/api-server/src/routes/studio.ts`:**
```ts
router.put("/studio/tasks/:id/status", requireAuth, async (req, res) => {
  const { status } = req.body;
  const user = (req as any).user;
  const task = await db.select().from(tasksTable).where(eq(tasksTable.id, id));

  // التحقق من الصلاحية
  const creativeRoles = ["designer", "photographer", "marketer"];
  const isCreative = creativeRoles.includes(user.role);
  const isAdmin = user.role === "admin";

  // المبدع يقدر فقط يغير: للتنفيذ→قيد التنفيذ، قيد التنفيذ→في المراجعة
  const creativeAllowed = ["in_progress", "in_review"];
  // المسؤول يقدر يغير: في المراجعة→معتمد/رفض، معتمد→منشور
  const adminAllowed = ["in_progress", "approved", "published"];

  if (isCreative && !creativeAllowed.includes(status)) {
    return res.status(403).json({ error: "غير مصرح" });
  }
  if (!isAdmin && !isCreative) {
    return res.status(403).json({ error: "غير مصرح" });
  }

  await db.update(tasksTable).set({ status }).where(eq(tasksTable.id, id));
  res.json({ success: true });
});
```

**في الواجهة `artifacts/kidspeak/src/pages/studio/index.tsx`:**
بعد تغيير الحالة، أضف:
```ts
qc.invalidateQueries({ queryKey: ["studio-tasks"], exact: false });
```

## 4 — ملاحظات للتنفيذ

- اجعل الصفحة الرئيسية للمبدع عند تسجيل الدخول هي **لوحة التحكم الجديدة** (`/studio`) وليس `/dashboard`
- في `App.tsx`: إذا كان المستخدم من `creativeRoles`، وجّهه لـ `/studio` عند login
- لا تُظهر للمبدع: التلاميذ، الأفواج، البرامج، التقييمات، الرواتب، المداخيل

## 5 — مرجع التصميم المقترح

استخدم هذا التصميم كمرجع بصري للتنفيذ:

```
┌─────────────────────────────────────────────────────┐
│  المحرك الإبداعي          [+ مهمة جديدة]            │
│  لوحة تحكم المبدع — أبريل 2026                      │
├──────────┬──────────┬──────────┬────────────────────┤
│ المهام   │بانتظار  │مكتملة   │ مستحقاتي           │
│ النشطة   │المراجعة │الشهر    │                    │
│    3     │    1    │    7    │   14,000 دج         │
├──────────┴──────────┴──────────┴────────────────────┤
│ مهامي                                               │
│ [للتنفيذ][قيد التنفيذ][في المراجعة][معتمد][منشور]  │
│                                                     │
│ • تصميم بوستر رمضان    [قيد التنفيذ] 2000دج [إرسال]│
│ • فيديو إعلاني         [في المراجعة] 5000دج         │
│ • حملة انستغرام        [معتمد]       3000دج [نشر]   │
├─────────────────────┬───────────────────────────────┤
│ أصولي المرفوعة      │ مستحقاتي                      │
│ [صورة][فيديو][pdf]  │ بوستر    2000دج ● مدفوع       │
│ [+ رفع]             │ فيديو    5000دج ● معلق         │
│                     │ حملة     3000دج ● معلق         │
└─────────────────────┴───────────────────────────────┘
```

**التصميم البصري:**
- الألوان: أزرق للمهام النشطة، برتقالي للمراجعة، أخضر للمكتملة، كورال للمستحقات
- الـ Tabs: خلفية رمادية فاتحة، الـ Tab النشط بخلفية بيضاء وحدود
- البطاقات الإحصائية: خلفية ثانوية بدون حدود
- الأزرار: حسب الحالة — أزرار صغيرة مع لون أزرق للإجراء الرئيسي
- اتجاه RTL عربي كامل
